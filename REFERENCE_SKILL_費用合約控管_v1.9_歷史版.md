# SKILL - 費用合約控管專案 v1.9

本 SKILL 適用於費用合約控管暨簽呈流程追蹤系統後續 UI、規格、HTML 原型與 Codex 開發交接。

## 1. 功能只增不減

除非使用者明確說不要，後續版本不得移除或降低既有功能。可以提出刪減或簡化建議，但不得自行套用。

## 2. 四角色入口

系統需維持四層角色入口：

- 處長：月度支出、待決策、已完成、差異摘要、案件履歷。
- 管理者 / 助理：案件管理、資料健檢、匯入回匯、金額驗證、全文搜尋、報表快照、動作紀錄、案件履歷。
- 主管：總覽、PM 時間線、簽呈看板、差異比較、明細、已完成、案件履歷。
- 承辦：我的待辦、新增案件、PM 時間線、簽呈看板、文件補件、核銷更新、案件履歷。

## 3. UI 原則

- 不在使用者畫面顯示內部討論文字、設計說明或規則註解。
- 左側導覽維持小型角色入口，不佔主要版面。
- 功能以 TAB 呈現，一個功能一頁，不把所有資訊往下堆。
- 每個摘要卡、數字、清單列、看板卡、來源卡都必須可下鑽。
- 下鑽路徑：摘要 → 條件清單 → 案件履歷 → PDF / Excel 來源 / 動作紀錄。
- 案件履歷應包含基本資料、PM 進度、簽呈流程、合約資訊、付款 / 核銷、金額來源、金額變更、歷史價格、附件、來源資料、動作紀錄。

## 4. 表格排序

任何表格式 UI 都必須支援欄位排序，包括主管明細、已完成、差異比較、管理者案件管理、動作紀錄、匯入批次、金額驗證、報表快照、承辦待辦、核銷更新等。

## 5. 統計平衡

同一篩選條件下，主狀態必須互斥且加總平衡：

```text
全部 = 未啟動 + 進行中 + 逾期 + 完成
```

風險標籤如卡關、缺文件、金額待確認、PDF 未索引、待決策可重複，需獨立顯示，不得混入主狀態加總。

## 6. 處長畫面

處長不需要 PM 線性圖主視圖。處長入口主打支出時間線、待決策、已完成、差異分析與案件履歷。

## 7. 主管畫面

主管需具備：

- PM 時間線
- 簽呈看板
- 已完成
- 差異比較：去年同月、去年同期、同案上次價格
- 明細清單

主管總覽、PM 時間線、看板、明細、差異比較需共用同一組篩選結果，數字要對得起來。

## 8. 管理者 / 助理案件權限

管理者與助理可新增、編輯、作廢案件，並查看動作紀錄。作廢取代永久刪除。所有新增、修改、作廢、還原、匯入、回匯、AI 確認、附件操作均需寫入 audit_logs。

## 9. AI 與來源確認

AI / OCR / 搜尋結果僅供輔助，不得直接寫入正式資料。搜尋結果與 AI 建議必須提供來源 Link。畫面需提示 AI 可能出錯，請人員開啟原始檔案確認。

## 10. 檔名規則

下載包命名格式：

```text
費用合約控管_<產出類型>_v版本_日期時間.zip
```

範例：

```text
費用合約控管_HTML子功能與規格_v1.9_20260608_0534.zip
```
