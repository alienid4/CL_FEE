# COMMON_SKILL 套用摘要 v2.0

本專案已套用以下共同 SKILL。

## 1. 功能只增不減

除非使用者明確要求，不得移除、降低、覆蓋既有功能。

## 2. UI 管理系統設計

- 角色入口清楚。
- 功能用 TAB 呈現。
- 一個功能一頁。
- 左側導覽不佔主要版面。
- 不顯示內部討論文字。
- 所有表格可排序。

## 3. Dashboard 下鑽與統計平衡

- 所有摘要數字可下鑽。
- 主狀態互斥。
- 總件數等於主狀態加總。
- 風險標籤可重複但不可混入主狀態加總。

## 4. 資料可信與來源追溯

重要資料需追來源：Excel、PDF、系統、人工、AI、報表快照。

## 5. AI 輔助與人工確認

AI 只能輔助，不作為最終依據。重要欄位需人工確認。

## 6. 匯入 / 回匯

匯入需暫存、檢核、確認。回匯不得直接覆蓋正式資料。

## 7. 權限與動作紀錄

新增、修改、作廢、匯入、回匯、附件、AI 確認都需記錄。

## 8. 報表快照

報表產出後需保存版本，後續資料異動不得覆蓋。

## 9. 文件附件與全文搜尋

需支援附件管理、PDF / Excel 來源 Link、文字型 PDF 搜尋。

## 10. Codex 開發規範

本案 v2.0 技術規範已改為 Python + MSSQL；共同規範中的「固定 Node.js」不再適用本案。
