# 怎麼用：哪台機器、點哪個檔

> `update` 和 `upload` 只差兩個字母，方向卻相反，很容易點錯。這張表就是為了不用再想。

## 一句話記法

- **upl**oad = **上**傳（你的電腦 ➜ GitHub）
- **upd**ate = **更**新下來（GitHub ➜ 對方電腦）

---

## 你的開發機（改程式的這台）

資料夾：`AI LAB\CL_FEE`

| 要做什麼 | 點這個 |
|---|---|
| 改完程式，推上 GitHub | **`upload.bat`** |

`upload.bat` 會依序：列出這次改了哪些檔（幾個檔、幾行）➜ 幫你打版本號 ➜ 你確認 ➜
自動同步版本號到四個位置並寫 CHANGELOG ➜ 跑完整測試 ➜ commit ➜ push。

- 跑一次約 **5 分鐘**（因為要跑 235 個測試），中途別關視窗
- 取消的話**不會動到任何檔案**
- 偵測到 `xlsx` / `db` / `.env` 會擋下來，要打 `YES` 才放行（CL_FEE 是公開 repo）

---

## 使用者 / 助理的電腦（只用系統、不改程式）

資料夾：`notebook-package`（整個複製過去，不能只複製 .bat 檔）

| 要做什麼 | 點這個 |
|---|---|
| 拿最新版程式 | **`update.bat`** |
| 開始使用系統 | **`service.bat`** ➜ 按 `1` |
| 停止 / 重開 | **`service.bat`** ➜ 按 `2` / `3` |
| 出問題要回報 | **`service.bat`** ➜ 按 `6` 產生診斷報告，把檔案傳回來 |

**標準流程**：先 `update.bat`，再 `service.bat` 按 `3` 重新啟動，然後在瀏覽器按
`Ctrl + Shift + R` 強制重新整理。

網址：**http://127.0.0.1:8888**

---

## 常見狀況

**「更新了但畫面沒變」**
按 `Ctrl + Shift + R`。一般的 F5 會用瀏覽器快取，看到的還是舊畫面。

**「點下去視窗就不見了」**
v0.22.0 之後不會了 —— 現在一定會停住並顯示原因。如果還是發生，看
`notebook-package\logs\service_error.log`。

**「怎麼登入都說密碼錯」**
多半是 `notebook-package\.env` 不存在，密碼根本沒被載入。`service.bat` 現在會自動建立。

**`update.bat` 不會更新 `service.bat` 本身**
它只同步 `app\` 和 `tests\`。控制台腳本改版時要手動複製 `service.bat` 與 `service.ps1`
過去（或整個資料夾重新複製一次）。
