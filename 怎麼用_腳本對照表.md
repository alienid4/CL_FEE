# 怎麼用：哪台機器、點哪個檔

> 檔名要讓人看一眼就知道方向與用途，不用先讀文件。這張表記錄目前的命名。

## 一句話記法

- **upload** = 上傳（你的電腦 ➜ GitHub）
- **download** = 下載（GitHub ➜ 對方電腦）
- **demo_** 開頭 = 展示／試用用的，不是正式入口

（舊名 `update.bat` 已於 v0.24.0 改為 `download.bat`——`update` 和 `upload` 只差兩個字母、
方向卻相反，太容易點錯。舊檔保留為轉接檔，點了會自動轉呼叫 `download.bat`。）

## 你的開發機（改程式的這台）

資料夾：`AI LAB\CL_FEE`

| 要做什麼 | 點這個 |
|---|---|
| 改完程式，推上 GitHub | **`upload.bat`** |

`upload.bat` 會依序：列出這次改了哪些檔（幾個檔、幾行）➜ 幫你打版本號 ➜ 你確認 ➜
自動同步版本號到四個位置並寫 CHANGELOG ➜ 跑完整測試 ➜ commit ➜ push。

- 跑一次約 **5 分鐘**（因為要跑 235 個測試），中途別關視窗
- 取消的話**不會動到任何檔案**
- 偵測到 `xlsx` / `db` / `.env` 會擋下來，要打 `YES` 才放行（CL_FEE 是公開 repo）

## 使用者 / 助理的電腦（只用系統、不改程式）

資料夾：`notebook-package`（整個複製過去，不能只複製 .bat 檔）

| 要做什麼 | 點這個 |
|---|---|
| 拿最新版程式 | **`download.bat`** |
| 開始使用系統 | **`service.bat`** ➜ 按 `1` |
| 停止 / 重開 | **`service.bat`** ➜ 按 `2` / `3` |
| 出問題要回報 | **`service.bat`** ➜ 按 `6` 產生診斷報告，把檔案傳回來 |

**標準流程**：先 `download.bat`，再 `service.bat` 按 `3` 重新啟動，然後在瀏覽器按
`Ctrl + Shift + R` 強制重新整理。

網址：**http://127.0.0.1:8888**

## 常見狀況

**「更新了但畫面沒變」**
按 `Ctrl + Shift + R`。一般的 F5 會用瀏覽器快取，看到的還是舊畫面。

**「點下去視窗就不見了」**
v0.22.0 之後不會了 —— 現在一定會停住並顯示原因。如果還是發生，看
`notebook-package\logs\service_error.log`。

**「怎麼登入都說密碼錯」**
多半是 `notebook-package\.env` 不存在，密碼根本沒被載入。`service.bat` 現在會自動建立。

**「點下去說找不到 Python，但我明明裝過」**
Windows 內建一支微軟商店的空殼 `python.exe`（在 `WindowsApps\` 底下），執行後立刻結束、
什麼都不印，但 `where python` 找得到它。v0.25.0 之後的腳本會實際執行 `--version` 驗證，
自動跳過空殼去找真的。若訊息說完全找不到，那就是真的沒裝，到 python.org 裝 3.11 以上，
安裝時勾選「Add python.exe to PATH」。

## 還有一支 demo_start.bat，那是展示用的

`demo_start.bat`（2026-07-21 前叫 `start.bat`）是 **8025 埠的試用模式**：用 `data\preview.db`、
免密碼登入，跟正式系統是**兩套不同的資料**。要用系統一律 `service.bat`（8888 埠）。

改名的原因：叫 `start` 看起來就是「開始」，最容易被誤點；點下去看到的是空的試用資料庫，
很像「資料全不見了」。加 `demo_` 前綴，看檔名就知道是什麼，不用先讀文件。

Python 偵測問題已於 v0.26.0 修好。但 `data\preview.db` 若不存在仍會自動建一個空的，
所以它只適合展示。**這支不會被 `download.bat` 同步，也不放進給部署機器的包。**
