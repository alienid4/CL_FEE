# PROJECT_SKILL_費用合約控管 v2.0

## 1. 專案名稱

正式名稱：費用合約控管系統  
完整說明名稱：費用合約控管暨簽呈流程追蹤系統  
簡稱：費控系統  
英文：FeeContractControl / FCCS

## 2. 專案專用角色

本專案固定四角色入口：

1. 處長：月度支出、待決策、已完成、差異摘要。
2. 管理者 / 助理：資料健檢、案件管理、匯入回匯、金額驗證、全文搜尋、報表快照、動作紀錄。
3. 主管：PM 時間線、簽呈看板、差異比較、明細、已完成、案件履歷。
4. 承辦：我的待辦、新增案件、PM 時間線、簽呈看板、文件補件、核銷更新、案件履歷。

## 3. 本專案不可刪除的功能

除非使用者明確指示，以下功能不得移除：

- 四角色入口。
- 各角色 TAB。
- 所有摘要下鑽。
- 表格排序。
- 案件新增 / 編輯 / 作廢。
- 動作紀錄。
- 金額驗證。
- Excel 匯入 / 回匯。
- PDF / Excel 來源 Link。
- AI 建議與人工確認分離。
- 報表快照。
- WBS 進度。

## 3.1 系統名詞規則

系統畫面、欄位、TAB、按鈕與報表皆使用正式名稱，不使用口語或簡寫。

- 使用 `預算承諾金額`，不使用 `已承諾` 或 `已占用預算`。
- 使用 `計畫外採購`，不使用 `臨採` 或 `臨時採購`。
- 使用 `已支出`，不使用 `已花`。
- 若因系統設計需要建立自訂名詞、改稱既有口語或重新命名流程，必須同步維護獨立的名詞備註文件，說明名詞用意、對應既有 Excel / 人工作業、避免的誤解與需求方確認狀態。
- 對外給需求方確認的文件與 HTML 不得只出現自訂名詞，需能追到名詞備註文件。

## 3.2 Excel 正式交付規則

本系統不得被設計成只看不能交付的查詢系統。最後繳交外部或最後單位的正式格式仍為 Excel。

- 系統定位：Excel 是正式交付格式，系統是資料控管、檢核、查詢、追蹤與匯出工具。
- 標準流程：`Excel 匯入 → 系統暫存 / 檢核 / 人工確認 → 系統追蹤與補狀態 → 匯出正式 Excel → 繳交最後單位`。
- 匯入時需保留原始 Excel 欄位、檔名、Sheet、Row、匯入批次與版本。
- 正式匯出 Excel 必須維持最後單位要求的原格式，不得任意新增欄位、改欄名、改欄序、改 Sheet 或破壞公式 / 樣式。
- 系統新增或維護的控管欄位若原範本已有對應欄位，才回填到正式匯出 Excel；若原範本沒有欄位，資料留在系統 DB、稽核紀錄或內部查核匯出，不放入正式繳交 Excel。
- 匯出 Excel 需支援最後單位要求的欄位順序、欄名、月份範圍、公式、合併儲存格、格式與樣式。
- 匯出前需做必要檢核，例如必填欄位、費用核銷月份、金額、請購編號、核銷編號與必填附件狀態。
- 不做 Excel 與系統雙邊任意覆蓋；若有回匯修正，必須先進暫存、差異比對與人工確認。

## 3.3 年度預算版本管理規則

`115年預算範本.xlsx` 這類年度預算範本屬於正式預算資料來源，需納入版本管理。

- 每次匯入年度預算 Excel 都建立一個預算版本，不以檔案修改時間直接覆蓋既有版本。
- 版本狀態至少包含：`待確認`、`目前正式版本`、`歷史版本`、`作廢`。
- 同一年度同一預算類型只能有一個 `目前正式版本`。
- 主管畫面預設讀取 `目前正式版本`，但需可追查歷史版本。
- 系統需保留每版原始 Excel 檔、檔案 hash、匯入人、匯入時間、Sheet、Row 與解析後資料。
- 新版匯入時需比對前一正式版本，列出新增、刪除、金額變更、負責窗口變更、類別 / 必要性變更、執行計畫變更與備註變更。
- 年度預算正式匯出需維持原範本格式，版本差異與系統追蹤資料留在系統畫面或內部查核匯出，不放入正式繳交 Excel。
- 預算資料不得只存單一年度，必須同時記錄 `預算編列年度` 與 `預算執行年度`。
- 例如 `115年預算範本.xlsx` 在畫面與資料上應呈現為 `114編列 / 115執行`，避免誤判為 115 編列。

## 3.4 A-G 流程 Dashboard 規則

A-G 流程是老闆 / 主管的主視圖，用來由上而下看預算、專案、簽呈、成案 / 未成案、核准、合約、付款之間是否斷點。A-G 不得取代既有 Excel-first 作業設計、業助工作台與前版已確認功能；專案、預算、支出費用、每月預計上簽、合約資訊應歸納到 A-G 節點內，並保留圖表與明細下鑽。若 A-G 畫面讓業助作業看起來像重新開荒，應回到既有 Excel 作業與上一版功能結構。

老闆 / 主管第一眼仍需能看到流程風險，但系統主架構需以既有 Excel 作業、角色工作台與實際操作功能為主。

流程固定為：

- A 年度預算。
- B 專案立案。
- C 簽呈送審。
- D1 成案。
- D2 未成案。
- E 核准結果。
- F 合約 / 請購。
- G 付款核銷。

每一個流程節點都需顯示可不點擊即可看到的主數字與風險數字。每一個節點也需保留自己的正式編號，例如預算項目編號、專案編號、簽呈編號、成案編號、未成案紀錄編號、核准紀錄編號、合約編號、請購編號、付款追蹤編號與核銷編號。

`Case_ID` 是系統內部串接主鍵，不取代各流程正式編號。

## 4. 處長規則

處長不主打 PM 線性圖。處長畫面重點：

- 本月 / 次月預計支出。
- 支出時間線。
- 待決策。
- 已完成。
- 金額變更。
- 差異分析。
- 案件履歷與來源。

## 5. 主管規則

主管需看流程與風險：

- PM 時間線。
- 簽呈看板。
- 明細。
- 已完成。
- 差異比較：去年同月、去年同期、同案上次價格。
- 狀態統計平衡。
- 可查詢案件、預算、支出費用、每月預計上簽與合約。
- 可新增主管交辦、確認結果、退回補件與風險標記。
- 不直接新增或修改合約 / 費用 / 預算主檔，除非使用者明確改變權限設計。

## 6. 管理者 / 助理規則

管理者 / 助理是主要操作角色：

- 新增案件。
- 編輯案件。
- 查詢、新增、修改合約續約資料。
- 查詢、新增、修改費用項目、付款明細、上簽清單與來源關聯。
- 查詢與更新每月費用付款狀態追蹤。
- 逐案設定付款附件必要性，不將附件必填規則寫死。
- 作廢案件，不做永久刪除。
- 匯入 Excel。
- 回匯 Excel。
- 檢查資料品質。
- 金額驗證。
- 報表產出。
- 查看動作紀錄。

所有操作需寫入 audit_logs。

## 6.1 費用付款狀態追蹤規則

- 付款追蹤主月份使用 `費用核銷月份`。
- 每個月都要能追蹤付款狀態。
- 處理進度固定包含：`發票尚未收到`、`資料蒐集中`、`可預備上簽`、`款項簽核中`、`款項已簽准`、`提交會計部(結案)`。
- 當處理進度為 `款項簽核中` 時，`核銷編號` 必填。
- `發票尚未收到` 需通知承辦與助理。
- 提醒方式包含系統畫面提醒與 Email。
- 狀態到 `提交會計部(結案)` 即結束，不追蹤實際付款完成。
- 附件必要性由助理依每筆案件設定為 `必填` 或 `選填`。
- 常見附件類型包含：發票、費用核銷申請書、簽核簽擬單、合約、費用分攤表、維護紀錄。

## 7. 承辦規則

承辦必須具備：

- 我的待辦。
- 新增案件。
- 更新自己案件狀態。
- 補文件。
- 核銷更新。
- 查看案件履歷。
