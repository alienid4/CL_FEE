# 一次性開發提示詞 v2.1 產品介紹

這套 **一次性開發提示詞 v2.1**，不是一般叫 AI 寫 code 的 prompt。

它是一套給 AI 開發者用的 **多 Agent 自動開發指揮包**。

## 痛點故事

很多 AI 開發者一開始都很興奮：

```text
我叫 Codex / Claude 幫我做一套系統，它很快就生出畫面和程式。
```

但真正痛苦在後面。

功能一多，AI 開始跑偏；修 A 壞 B；測試沒跑；需求要一直重講；多 Agent 各說各話；Coder 說完成，但沒人知道到底測了沒；Reviewer 只是口頭稱讚；最後 AI 開發者變成「人肉 PM + QA + 架構師 + 稽核員」。

90% 時間不是在創造新系統，而是在追 bug、補規格、盯測試、管 Agent。

## v2.1 解決什麼

v2.1 把這件事變成一套固定流程：

- 先訪談，不急著亂寫 code。
- 產出 MVP 契約，先定義第一版做什麼、不做什麼。
- 自動拆切片。
- 用多 Agent 分工：Architect、Coder、Tester、Reviewer、Auditor。
- 小問題 AI 自己修。
- 重大問題才問人。
- 用 Speed Lane 壓榨效率。
- 用 Focus Gate 防止跑偏。
- 用 Audit Gate 防止自說自話。
- 用 Demo Gate 確保最後交付可展示 MVP。

## 核心優勢

### 1. 一包式，不用自己組裝

不是主包、副包、控制台、規則分開買。

它像 Notebook，一包開機可用。

### 2. CIO Build Mode

使用者只要說「我要開發一套 XX 系統」，AI 就會先訪談、整理 MVP 契約，再開始開發。

不再是一句話亂生 code。

### 3. 多 Agent 開發小隊

它不是單一 Agent 自說自話，而是讓 Architect、Coder、Tester、Reviewer、Auditor 各自負責不同階段。

### 4. 糾察隊互評機制

Coder 不能自己宣布完成。

Tester 要驗證，Reviewer 要查風險，Auditor 要看證據。

沒證據不能 pass。

### 5. 壓榨開發效能

小修走 Fast Lane，不用每次啟動重流程。

一般功能走 Standard Lane。

高風險才走 Release Lane。

快的地方快，該穩的地方穩。

### 6. 可連續開發數小時

每輪都有狀態、下一步、測試、audit。

AI 不需要每次重新理解全部背景，可以持續推進。

### 7. 自己修 bug，重大才問人

明確 bug、測試失敗、小範圍修正，AI 自己處理。

只有正式資料、DB、權限、部署、金額邏輯這類重大事項才問使用者。

### 8. Focus Gate 防止跑偏

每輪先確認目標、非目標、允許改什麼、禁止改什麼。

避免 AI 做一做跑去修不相關工具或開新支線。

### 9. Audit Gate 防止假完成

每輪都要有測試與證據。

不是 AI 說完成就完成。

### 10. Demo Gate 讓交付可驗收

最後交付不是一堆 code，而是可跑版本、完成清單、測試結果、已知限制與下一步。

## 一句話介紹

> 一次性開發提示詞 v2.1 是一套讓 Codex / Claude 從「寫 code 助手」升級成「多 Agent 自動開發小隊」的一包式指揮包，能連續開發、自己修 bug、互相稽核，最後交付可驗收 MVP。

## 更有力的說法

> 它的價值不是讓 AI 寫得更快一點，而是讓 AI 開發變成可控、可驗證、可連續推進的自動化流程。使用者不再當人肉監工，而是回到方向決策者。
