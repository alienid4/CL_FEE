# CL_FEE 未來待辦

pilot 已可用。以下為未來項目，多數需使用者決策或外部資源（非阻礙 pilot）。

## 需使用者提供資源
- **催辦 email 寄送**：提供 `SMTP_HOST` + `EMAIL_MAP`（帳號→email），即可從站內預覽升級為真寄。
- **真實 Excel 欄位**：提供預算/專案/簽呈/請購的真實表頭，微調欄位（ensure_column 便宜擴充）。

## 需使用者拍板方向（重大事項，屬絆線）
- **遷 MSSQL**：正式環境資料庫；換連線層 + SQL 方言。
- **AD / SSO**：取代本機帳號。
- **全單位正式上線**：目前為 pilot；正式上線範圍、備援、稽核保存政策需確認。
- **金額硬規則**：是否硬擋負數金額、合約金額 > 案件金額、超支硬阻擋（目前為提醒）。
- **匯入撞到同名資料的處理方式**：目前（專案匯入為例）比對到同一識別鍵（組別＋專案名稱）就直接自動覆蓋更新，不會先給使用者看 diff 或問要不要蓋。使用者 2026-07-11 反饋希望改成「先預覽、問哪個為準，使用者確認才覆蓋」，但要做到什麼程度（哪些匯入模組要套用、預覽介面長怎樣）還沒討論，先記著。

## 可自主做的強化（未來切片）
