# 資料提供限制與分階段開發

## 決策

發票 PDF 原檔、電子發票檔、多筆發票 Excel 清單、完整簽呈 PDF、完整合約 PDF 等資料可能包含敏感資訊，第一版不要求提供。

系統採兩階段開發：

1. 第一版：先完成系統本體、資料模型、畫面、查詢、下鑽、Case 360、舉證位置與手動維護能力。
2. 第二版：等有脫敏資料、測試資料或正式授權資料後，再開發自動匯入、PDF 解析、電子發票解析與批次比對。

## 第一版範圍

第一版要先能讓人操作、看懂、追蹤。

包含：

- 八項控管看板。
- 12 項核心功能入口。
- 全文檢索。
- Dashboard 數字下鑽。
- Case 360。
- 時間線。
- 處理優先矩陣。
- 負責人視角。
- 廠商視角。
- 合約、簽呈、付款、發票的資料欄位預留。
- 附件 / PDF 的登錄位置。
- 舉證鏈欄位。
- Excel 匯入 mapping 設計。
- CMDB 關聯欄位預留。

第一版資料來源：

- 使用者提供的截圖樣本。
- 可公開或可脫敏的測試資料。
- 人工新增 / 維護資料。
- 少量模擬資料。

第一版不做或只做預留：

- 真實發票 PDF 自動解析。
- 電子發票批次匯入。
- 大量簽呈 PDF 自動解析。
- 合約 PDF 自動解析。
- 真實敏感附件上傳。
- 真實 CMDB 串接。

## 第二版範圍

第二版在資料可提供後再做。

包含：

- 發票 PDF / 電子發票檔解析。
- 多筆發票 Excel 匯入。
- 簽呈匯出檔或 PDF 解析。
- 合約 PDF 解析。
- 附件批次上傳。
- 欄位自動 mapping。
- 匯入批次差異比對。
- 金額、稅額、付款排程自動核對。
- CMDB 查詢或匯入。

第二版需要的資料：

- 脫敏後的發票樣本。
- 脫敏後的簽呈樣本。
- 脫敏後的合約樣本。
- 多筆測試 Excel。
- 欄位對照說明。
- 正式權限與資料使用規範。

## 資料安全原則

- 不要求使用者提前提供敏感原檔。
- 第一版以欄位設計與流程設計為主。
- 若需測試資料，優先使用脫敏資料。
- 真實資料接入前需確認權限、保存位置、遮罩規則與稽核紀錄。
- 系統要保留來源欄位，但不強迫第一版一定要有原始 PDF。

## 驗收方式

第一版驗收：

- 使用者可以操作系統流程。
- 可以查案件、看 Case 360、看時間線、看下鑽。
- 可以看到發票、簽呈、合約的預留欄位與關聯位置。
- 可以人工建立樣本資料。
- 可以清楚知道哪些欄位未來由第二版匯入。

第二版驗收：

- 能匯入真實或脫敏批次資料。
- 能保留來源檔、頁碼、列號與欄位。
- 能自動比對付款、合約、發票與簽呈。
- 能標出缺漏、金額差異與待人工確認項目。
