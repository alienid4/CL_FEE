# CL_FEE Pilot 試用測試清單（照著點就行）

## 開始前

- 網址：`http://localhost:8010`，密碼一律 `1qaz@WSX`（測試用）。
- 帳號：`ap01`＝處長/CIO（唯讀）、`ap02`＝主管/助理（可讀寫看全部）、`ap03`＝承辦（只看自己）。
- **只測下面列的東西。左側標「尚未啟用」的模組（預算/專案/簽呈/請購/核准）先不要點，那些還沒做。**
- 每一步都寫了「應該看到什麼」，對得上就是正常。

---

## A. 登入與角色（約 3 分鐘）

| 做什麼 | 應該看到 |
|---|---|
| 用 `ap02` 登入 | 進到主畫面，左上顯示「登入身分：主管/助理（ap02）」 |
| 登出，改用 `ap01`（CIO）登入，到「新增案件」按儲存 | **被擋、存不進去**（CIO 只能看，不能改） |
| 登出，改用 `ap03`（承辦）登入 | 案件清單**只看得到自己的**（一開始可能是空的） |

## B. 案件流程 — 核心（約 5 分鐘，用 `ap02`）

| 做什麼 | 應該看到 |
|---|---|
| 到「新增案件」，填：編號、名稱、負責人、金額、狀態選「審核中」、**備註**、**下一步**，按儲存 | 案件出現在下方清單 |
| 看畫面上方「**需處理案件（待辦）**」那塊 | 剛才那筆（審核中/有下一步）**出現在待辦裡**，顯示狀態/備註/下一步 |
| 點待辦清單裡那筆案件 | **跳到編輯**，表單自動帶入資料 |
| 改一下備註，按儲存 | 清單、待辦、CIO 案件表都更新 |
| 把某案件狀態改成「已核准」再存 | 它**從待辦清單消失**（已核准就不算待辦了） |

## C. 月度支出（約 3 分鐘，用 `ap02`）

| 做什麼 | 應該看到 |
|---|---|
| 新增一筆「合約」（在合約模組） | 合約清單出現該筆 |
| 在該合約下新增幾筆「付款」，不同月份、金額，有的狀態設 `closed` | 付款清單出現 |
| 看「**月度支出彙總**」表 | 依月份加總：總額 = 該月付款相加、`closed` 的算「已付」、其餘算「待付」 |

## D. 權限隔離 — 最重要（約 3 分鐘）

| 做什麼 | 應該看到 |
|---|---|
| 用 `ap03`（承辦）登入，新增一個案件 | 該案件自動歸 ap03；ap03 看得到 |
| 登出，用 `ap02`（主管）登入 | **看得到 ap03 剛建的案件**（主管看全部） |
| 再用 `ap03` 登入 | **只看得到自己的**，看不到別人建的案件 |
| 用 `ap03` 試著點左側「資料檢核」或稽核 | 承辦**不能用**（無權限） |

## 測到怪怪的怎麼辦

- **記下來**：哪個帳號、點了什麼、預期看到什麼、實際看到什麼。截圖更好。
- 把這幾點給我，我就能定位、修、補測試。
- 系統健康檢查：瀏覽器開 `http://localhost:8010/health` 應回 `{"ok": true, ...}`。

## 這份清單沒有的 = 還沒做

預算、專案、簽呈、請購、核准、來源舉證鏈——這些是**未來功能**，pilot 階段不測。等你們用核心流程有回饋，再決定要不要蓋。
