## 五、付款與合約關聯

此發票可透過以下資訊關聯：

| 關聯鍵 | 樣本內容 | 可關聯對象 |
|---|---|---|
| 訂單號 | POE02026010034 | 請購 / 採購單 |
| 品名 | RedHat OpenShift 軟體維護合約 - 第一期款 | 合約、付款排程 |
| 期別 | 第一期款 | 付款排程 |
| 金額 | 512,500 / 488,095 + 稅 | 付款金額、合約期款 |
| 廠商統編 | 53916262 | 廠商主檔 |
| 買方統編 | 27236155 | 公司主檔 |

建議發票關聯狀態：

- 待關聯：尚未找到付款排程或合約。
- 已關聯付款：已找到對應付款排程。
- 金額一致：發票總額與付款排程金額一致。
- 金額差異：發票總額與付款排程不同。
- 缺付款：有發票但沒有付款排程。
- 缺發票：付款排程到期但尚未收到發票。

## 六、系統功能對應

此發票樣本可支援：

- 發票主檔。
- 發票明細。
- 發票稅額檢核。
- 廠商視角。
- 付款排程關聯。
- 合約期別關聯。
- 6 月 / 指定月份應開發票查詢。
- Case 360 發票分頁。
- 舉證鏈：發票號碼、頁碼、金額、訂單號、統編。
- 全文檢索：發票號碼、廠商、統編、訂單號、品名。

## 七、仍需補充的原始資料

截圖可做欄位設計；若要自動匯入，建議提供：

1. 發票 PDF 原檔或電子發票檔。
2. 多筆發票清單 Excel。
3. 發票與付款排程的關聯規則。
4. 發票與採購單 / 訂單號的關聯規則。
5. 若有折讓、作廢或重開發票，也需提供樣本。

## 八、解析規則建議

未來發票解析時，建議優先抽取：

- 發票號碼。
- 發票日期。
- 買方名稱與統編。
- 賣方名稱、統編、地址。
- 品名。
- 數量、單位、單價、金額。
- 銷售額。
- 稅額。
- 總額。
- 訂單號 / 採購單號。

每一筆抽取結果都要保留：

- PDF / 圖檔名稱。
- 頁碼。
- 原文片段或座標。
- 抽取欄位。
- 抽取信心或人工確認狀態。
