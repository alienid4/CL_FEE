## CMDB 關聯預留

CMDB 分兩階段處理：

第一階段先完成費用合約控管本體，但在 Case、專案、合約、付款與文件資料中保留 CMDB 關聯欄位。

第二階段再依實際來源接 CMDB 查詢或匯入，來源可能是：

- CMDB API。
- CMDB Excel 匯出。
- CMDB CSV 匯出。
- 人工關聯表。

若未來先用 Excel / CSV 匯入 CMDB，建議提供欄位：

- CI ID。
- CI 名稱。
- 系統名稱。
- 服務名稱。
- 環境別。
- 所屬部門。
- 系統負責人。
- 服務負責人。
- 狀態。
- 最後更新時間。
- 與 Case / 合約 / 專案的關聯鍵。
