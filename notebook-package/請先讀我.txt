CL_FEE 完整包
（版本與日期見檔名）

這份是用 mail 寄給你的完整程式包。照下面做就好，大概五分鐘。


【一、安裝】

1. 找到你電腦上的 notebook-package 資料夾
   （裡面應該有 service.bat）

2. 把這個包裡的東西「全選、複製」，貼到 notebook-package 裡面

3. Windows 會問要不要取代，選「取代目標中的檔案」

4. 完成


【二、最重要：不要先刪掉舊資料夾】

請用「貼上覆蓋」，不要把整個舊的 notebook-package 刪掉再貼。

因為你資料夾裡有兩樣東西，這個包裡「故意沒有」：

      data\     <- 你的資料庫，系統的所有資料都在裡面
      .env      <- 你的設定檔

刪掉重貼會把它們一起殺掉，而且這個包裡沒有可以還原的東西。
用覆蓋的話它們會原封不動保留。


【三、裝好請做這件事，然後回信給我】

1. 雙擊 service.bat

2. 選 6（產生診斷報告）

3. 到 logs\ 資料夾裡找剛產生的報告檔，用 mail 當附件寄回來

   不管成功還是失敗都請寄。這份報告會告訴我你電腦上的 Python
   到底是哪一個版本、裝在哪、套件裝了沒、網路通不通。
   有這份我才知道下一步要給你什麼，沒有的話只能猜。

   如果連 service.bat 都打不開，請改寄 logs\service_error.log。


【四、這個包裡只有四樣東西要管】

      service.bat                <- 唯一的入口，開始使用系統
      service.ps1                <- service.bat 會用到，不要動它
      app\                       <- 程式本體
      requirements-runtime.txt   <- 系統需要的套件清單

其他的 data\ 和 .env 會在第一次啟動時自動產生，你不用管。


【五、跟你之前那版的差別】

一、視窗不會再閃一下就關掉

  舊版啟動失敗時整個視窗直接消失，什麼都看不到。
  新版不管失敗在哪都會停住、印出原因，並把錯誤寫進
  logs\service_error.log。

二、要裝的套件從 7 個減到 3 個

  舊版會連測試工具一起裝，其中一個叫 playwright，
  它裝完還會再去網路上抓瀏覽器，在公司網路多半會失敗——
  而你的電腦根本用不到它。
  新版只裝真正需要的三個：fastapi、uvicorn、openpyxl。

三、修好 Excel 匯入匯出

  舊版少裝了 openpyxl 這個套件，它是 Excel 匯入匯出要用的。
  少了它系統照樣開得起來，只有真的按下匯入或匯出才會失敗，
  所以一直沒被發現。這版補上了。
  新版還會在啟動前先檢查它，少了會直接講，不會等你用到才炸。


【六、注意】

一、公司網路連不到 GitHub，所以「自動更新」在你那邊不能用

  這個包裡沒有放 download.bat 和 update.bat。
  那兩支是去 GitHub 抓最新程式的，你們公司網路擋住了 GitHub，
  點了只會失敗。放進來只會讓你白試一場。

  以後要更新，還是我用 mail 寄新的包給你，做法跟這次一樣。

二、不要用記事本另存 service.ps1

  這個檔含中文，記事本另存新檔會改掉編碼，
  之後畫面會變亂碼或直接跑不動。


【有問題】

把 logs\ 裡的檔案寄回來就好，不用自己判斷是什麼問題。
