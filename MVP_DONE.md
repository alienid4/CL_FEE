# CL_FEE MVP 完成清單

對照使用者原始需求，逐項標示完成狀態。基準：HEAD、119 pytest + 25 E2E 全過。

## 核心角色與權限
- [x] **CIO（ap01）** 極簡「決策總覽」+ 逐層下探；其他模組完全隱藏（非灰色）
- [x] **主管/助理（ap02、ap04）** 全功能
- [x] **承辦（ap03）** 只看自己案件及其下的合約/付款/專案/請購/簽呈
- [x] 承辦禁用稽核/匯入/開發者控制台

## 雙人複核（金流正確性）
- [x] 狀態機：草稿 →〔送出〕待複核 →〔另一人核准〕已核准
- [x] 鐵則 A：建立者不得核准自己的案件（前後端雙擋）
- [x] 鐵則 B：核准前不進 CIO 畫面/金額
- [x] 擋「直接設已核准」（只能走 /approve）
- [x] 主管「待我複核」佇列

## 八大功能模組（皆可 CRUD）
- [x] 案件管理　[x] 合約　[x] 付款　[x] 資料檢核
- [x] 預算　[x] 專案　[x] 簽呈　[x] 請購
- [x] 每筆可關聯案件（case_id），供 CIO 下探整條控管鏈

## CIO 決策總覽
- [x] 資金看板：下月應付 / 要準備的資金 / 本月應付 / 下月預算外
- [x] 圖表：計畫內 vs 預算外圓餅、未來 6 個月現金流長條
- [x] 下月要出的款清單（可點下探）
- [x] 下探顯示：對應預算/專案/簽呈/請購/合約/付款/文件
- [x] 預算外偵測（案件無對應預算）
- [x] 超支偵測（案件付款 > 對應預算）
