# Handoff Checklist

## 交接前

- [ ] `HANDOFF_INDEX.md` 已更新。
- [ ] `Memory.md` 已更新最近變更。
- [ ] `docs/current_state_snapshot.md` 已更新。
- [ ] 測試方式與最近結果已記錄。
- [ ] 已知風險與 workaround 已記錄。
- [ ] release / patch / install 方式已記錄。
- [ ] 敏感資料規則已確認。
- [ ] debug bundle 或診斷方式可安全執行。

## 交接時

- [ ] 接手者知道從哪份文件開始。
- [ ] 接手者知道哪些資料不能外傳。
- [ ] 接手者知道哪些操作需要人工核准。
- [ ] 接手者知道目前哪些功能可用、哪些功能不可用。
- [ ] 接手者知道下一個最小任務。

## 交接後驗收

- [ ] 接手者能說明系統用途。
- [ ] 接手者能跑出基本驗證。
- [ ] 接手者能指出高風險區域。
- [ ] 接手者能提出不破壞現況的下一步。
- [ ] 不需要原開發者反覆補充同一批背景資料。
