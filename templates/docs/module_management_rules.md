# Module Management Rules

每個模組都要能被管理，不能只有功能畫面。

## 每個模組至少要有

- 啟用 / 停用。
- 權限設定。
- 預設參數。
- 連線設定。
- 測試連線或健康檢查。
- 通知設定。
- 稽核紀錄。
- 匯入 / 匯出或備份策略。

## 設定位置

開發 A 模組時，A 模組的設定要放在管理介面的 A 模組底下。

不要把 A 模組設定散落在全域設定、環境變數、README 與程式碼多個地方。

## 模組狀態

模組狀態至少包含：

- enabled
- disabled
- readonly
- degraded
- maintenance

狀態要能被 UI、API 與健康檢查共同讀取。
