# Add-on Pack Strategy

主包只放所有專案都需要的基本功，附屬包放特定任務能力。這樣可以避免每個專案都背負過多文件與技能，也可以讓公司內部 GPT 依任務載入最少需要的規則。

## 分層

| 層級 | 內容 | 是否必裝 |
| --- | --- | --- |
| Core Kit | 專案基本功、交接、測試、安全、反偷懶、證據規則 | 必裝 |
| WBS Pack | WBS、里程碑、工作拆解、驗收清單 | 選裝 |
| Executive Pack | CIO / 客戶簡報、效益、風險、案例故事 | 選裝 |
| Security Pack | 黑箱、白箱、弱點掃描與敏感資訊檢查 | 選裝 |
| UI Governance Pack | UI 密度、tab、排序、全文檢索、數字下鑽 | 選裝 |
| DB Auth Pack | MSSQL、AD、Email 設定與測試 | 選裝 |

## 規則

- 附屬包可以依賴主包，主包不能依賴附屬包。
- 附屬包不能覆蓋主包的安全與驗證要求。
- 每個附屬包都要說明用途、使用時機、輸入、輸出與驗證方式。
- 附屬包若產生文件或簡報，必須避免敏感資訊。
