# Product UI Rules

企業內部工具的 UI 目標是讓使用者更快完成工作，不是展示功能有多少。

## 版面

- 不要大量留白。
- 一頁聚焦一個主要功能。
- 子功能用 tab 分開。
- 預設顯示下一步，不預設攤開巨大表格。
- 低頻功能收進「更多」、「進階」或可展開區。

## 表格

- 表格必須可排序。
- 表格要有搜尋與篩選。
- 需要大量資料時，預設不顯示全表。
- 使用者搜尋、篩選、點數字卡或按顯示全部後才展開。
- 列表要保留返回、清除條件與目前條件提示。

## 查詢與下鑽

- 必須有全文檢索。
- 數字卡必須可點擊下鑽。
- 下鑽後要能回到上一層。
- 下鑽結果要顯示篩選條件，避免使用者不知道自己看到什麼。

## 操作一致性

- 有新增，就要有修改與刪除或停用。
- 有批次操作，就要能明確看到選取數量與完成結果。
- 儲存後要顯示結果與下一步。
- 錯誤訊息要告訴使用者可以做什麼，不只顯示失敗。
