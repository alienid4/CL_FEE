# Data Quality Rules

用來定義資料匯入、清理、正規化與異常判斷規則。

## 重要欄位

| 欄位 | 必填 | 可空 | 正規化規則 | 錯誤處理 |
| --- | --- | --- | --- | --- |
| id | 是 | 否 |  |  |
| name | 是 | 否 | trim, alias mapping |  |
| status | 是 | 否 | 使用狀態字典 |  |

## 狀態字典

| 狀態 | 意義 | 是否進入正式流程 |
| --- | --- | --- |
| active | 使用中 | 是 |
| draft | 草稿 / 待確認 | 否 |
| disabled | 停用 | 否 |
| pending_delete | 待刪除 | 否 |

## 正規化

- 同義詞要集中管理，不要散落在 UI。
- 使用者常輸入錯字時，要先做 alias mapping。
- 不確定資料不要自動覆蓋正式資料，應標記為待確認。

## 匯入後檢查

- 總筆數。
- 成功筆數。
- 失敗筆數。
- 必填缺漏。
- 重複資料。
- 狀態不合法。
- 需要人工判斷的資料。
