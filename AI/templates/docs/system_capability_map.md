# System Capability Map

用來讓主管、維運與接手 AI 快速知道系統有哪些能力，以及每個能力背後的資料來源。

| 能力 | 使用者 | 資料來源 | 關鍵 API / job | 狀態 | 備註 |
| --- | --- | --- | --- | --- | --- |
|  |  |  |  | 可用 / 部分 / 未完成 |  |

## 能力分層

| 層級 | 意義 | 範例 |
| --- | --- | --- |
| L1 | 基本查詢與可視化 | 清單、搜尋、基本狀態 |
| L2 | 資料治理與批次處理 | 匯入、正規化、批次變更 |
| L3 | 深度分析或自動化 | 風險判讀、排錯、報表 |

## 不要混在一起

- 畫面展示。
- 資料治理。
- 自動化動作。
- 高風險寫入。

每種能力要能說清楚資料從哪裡來、誰能操作、失敗時怎麼回復。
