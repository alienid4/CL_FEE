# UI Workflow Principles

用來避免系統功能越做越多，但使用者越來越看不懂。

## 原則

- 預設顯示下一步，不預設攤開所有資料。
- 常用功能放前面，低頻功能收進「更多」或進階區。
- 使用者完成動作後，要明確看到結果與下一步。
- 表格過大時，預設收起，搜尋、篩選、點卡片或顯示全部後再展開。
- 批次處理不要預設攤開，進入清單後需要時才顯示。
- 備註與說明不要佔滿版面，可用提示圖示或可展開區。

## 每個畫面都要回答

- 我現在在哪裡？
- 我可以做什麼？
- 做完會去哪裡？
- 失敗時為什麼？
- 有沒有危險操作？

## UI 驗收

- 沒看文件也知道第一步。
- 新手不會卡在同一個按鈕反覆按。
- 大量資料不會一進畫面就壓垮可讀性。
- 狀態、數量、篩選結果一致。
