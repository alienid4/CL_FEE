# 費用合約控管系統 WBS 進度 v2.1

版本：v2.1  
日期：2026-06-10  
系統名稱：費用合約控管系統  
完整名稱：費用合約控管暨簽呈流程追蹤系統  
技術方向：Python FastAPI + Microsoft SQL Server + Web Service  

## 1. 文件目的

本文件用來作為費用合約控管系統的工作分解結構、進度追蹤、交付驗收與後續 Codex 開發依據。

本版先聚焦第一階段 MVP，目標是做出可用、可查、可驗收、可交接的內部 Web 系統。

## 2. MVP 範圍

### 2.1 第一階段必做

- 四角色入口：處長、管理者 / 助理、主管、承辦。
- 費用合約主檔 CRUD。
- 案件新增、編輯、作廢與動作紀錄。
- 月度支出預估。
- 簽呈流程追蹤。
- 合約續約管理。
- 核銷付款進度。
- 金額驗證中心。
- 金額變更管理。
- 歷史價格比較：去年同月、去年同期、同案上次價格。
- Excel 匯入暫存、檢核、人工確認。
- 匯出符合最後單位格式的正式 Excel。
- Excel 回匯差異比對，不直接覆蓋正式資料。
- PDF / 附件管理與來源 Link。
- 文字型 PDF 內容搜尋，可先不做掃描 OCR。
- 報表中心與報表快照。
- Swagger / OpenAPI 文件。
- WBS 進度文件與每次交付更新機制。

### 2.2 第二階段再做

- 掃描 PDF OCR。
- 會計 / 採購 / 正式簽核系統介接。
- AI 語意問答搜尋。
- 更完整 BI 圖表。
- 更細緻的權限矩陣。

| DB | MSSQL 主機、DB 名稱、帳號與權限尚待確認 | 影響連線與 migration | 先用 `.env.example` 與 mock / local config 開發。 |
| 資料 | Excel 來源檔欄位 mapping 尚未完成 | 影響匯入與主檔建模 | 先列 mapping 表，再做匯入。 |
| 權限 | 四角色權限細節尚未完整定義 | 影響 UI 與 API 授權 | 第一版先用角色入口與功能邊界，之後補權限矩陣。 |
| AI / PDF | AI 與 OCR 不可直接進正式資料 | 影響金額驗證流程 | 全部先進暫存與建議區，需人工確認。 |
| 回匯 | Excel 回匯可能覆蓋新資料 | 影響資料正確性 | 使用 systemId、version、exportBatchId、row_version 防衝突。 |
| 報表 | 後續資料異動可能影響已產出報表 | 影響稽核與追溯 | 報表必須建立 snapshot。 |
